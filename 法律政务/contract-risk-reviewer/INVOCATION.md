# Invocation

## 执行顺序

1. 读取 skill.json，确认输入、输出和边界。
2. 读取 SKILL.md，按任务需要加载其中指定的 reference 和 asset。
3. 盘点用户证据，分开记录事实、假设、缺口和真正阻塞的决策。
4. 先完成稳定且可证实的部分；不得为了文档完整而补造事实。
5. 按 SKILL.md 的 Output contract 交付，并用 eval/acceptance.md 自检。

## 失败处理

- 缺少非阻塞信息：继续完成可执行部分，并显式列出假设与下一证据。
- 缺少会改变结论的关键信息：指出具体缺口、影响和最低补充要求。
- 来源相互冲突：保留冲突，不擅自选取有利数字。
- 请求越过 skill 边界：完成安全范围内工作并说明需要的专业升级。
