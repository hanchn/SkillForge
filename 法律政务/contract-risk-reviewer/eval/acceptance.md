# Acceptance

## 必须通过

- [ ] 审阅方、合同类型、管辖与缺失附件已说明
- [ ] 问题均定位到条款并解释具体暴露
- [ ] 义务包含主体、动作、触发、期限、证据与后果
- [ ] 严重性与概率分开判断
- [ ] 高风险管辖问题明确升级给律师
- [ ] 事实、假设、推断、建议和未知项明确区分
- [ ] 输出符合 SKILL.md 的 Output contract

## 直接失败

- [ ] 未发生：不替代有执照律师的正式法律意见
- [ ] 未发生：不虚构管辖法律、市场惯例或可执行性
- [ ] 未发生：不把建议措辞描述成已获法律确认
- [ ] 未用空泛最佳实践替代当前任务证据
- [ ] 未隐藏会改变结论的重要信息缺口

## 人工抽查

- 随机选择一个关键结论，能追溯到输入证据或明确假设。
- 随机选择一个建议，能说明影响对象、执行条件和验收方式。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `contract_risk_register.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `obligation_register.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `negotiation_priorities.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
