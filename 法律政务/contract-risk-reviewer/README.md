# 合同风险审阅师

> skill id：contract-risk-reviewer
> 当前版本：1.0.1
> 产品状态：可用
> 所属分类：法律政务 / 合同 / 商业合同审阅

## 产品定位

以指定合同一方为视角，逐条提取义务、识别风险和缺失保护，并整理谈判与律师升级清单。

## 解决的问题

- 审阅只摘录条款，没有说明对指定一方的具体暴露
- 期限、触发、责任人和违约后果没有形成运营义务清单
- 缺失附件、交叉引用冲突和管辖法问题容易被忽略

## 适用场景

- 首轮审阅 NDA、服务、采购、SaaS、许可或数据处理合同
- 提取全部带期限和内部责任的义务
- 形成风险登记、谈判优先级和专业律师升级项

## 合规与权限

- 合规配置：`legal`（法律工作合规基线）。
- 执行前必须读取 `references/compliance-baseline.md`，完成司法辖区、数据、权限、授权、人工复核和审计检查。
- Skill 只能提供第一层规则约束，不能替代系统权限、真实授权、正式审批或专业法律意见。

## 不适用范围与边界

- 不替代有执照律师的正式法律意见
- 不虚构管辖法、可执行性、法定要求或市场惯例
- 建议措辞只能作为律师复核的谈判草案

## 输入

- 合同正文或文件与审阅方
- 可选管辖、交易目的、商业重要性和谈判底线

## 输出

- 合同风险登记表
- 义务清单
- 缺失保护和谈判优先级
- 专业律师升级清单与剩余风险

## 工作原理

1. 保留条款编号、定义和引用关系
2. 建立合同主题地图并识别缺失附件
3. 提取主体、动作、触发、期限、证据和后果
4. 检查定义、日期、顺位、救济和存续冲突
5. 按严重性和概率排序风险
6. 提出首选、妥协和升级路径

## 使用方式

1. 将整个 contract-risk-reviewer/ 目录交给支持 skill 的 AI 或复制到对应技能目录。
2. 让 AI 先读取 SKILL.md；README.md 用于人类理解产品能力和升级历史。
3. 按任务需要提供输入证据，执行后使用 eval/acceptance.md 验收。
4. 若修改能力、边界、输入输出或资源，应同步更新 README.md 的当前说明和版本记录。

## 验收标准

- 审阅方和关键假设明确
- 每项风险能定位到条款或缺失项
- 义务具备触发、期限、后果和内部责任
- 管辖和高风险问题正确升级

## 版本与升级规则

- 当前能力以本 README 和 skill.json 的版本为准。
- 每次能力、工作流、资源、输入输出或边界发生实质变化，必须提升版本并补充下方记录。
- 仅修正文案错字或链接时可不提升版本，但仍应保证产品说明与实际文件一致。
- 历史记录只写已发生变化，不把计划中的能力写成已经交付。
