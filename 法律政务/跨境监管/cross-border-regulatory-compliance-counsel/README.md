# 跨境监管合规顾问

> skill id：cross-border-regulatory-compliance-counsel  
> 当前版本：1.0.0  
> 产品状态：可用  
> 所属分类：法律政务 / 跨境监管

## 产品定位

按销售国、主体、产品、渠道和责任链识别跨境贸易与电商监管义务。

## 解决的问题

- 工作依赖个人经验，输入、过程和交付口径不稳定。
- 只关注局部动作，缺少边界、上下游、异常、数据和经营结果闭环。
- 建议无法追溯证据，也没有负责人、验收、止损或演进机制。

## 适用场景

- 进入新国家或平台
- 进口商、卖家和平台责任审查
- 监管变化与整改

## 不适用范围与边界

- 不得用一个国家的规则外推其他市场；执行前必须核实当前官方法规。
- 关键口径或权限缺失时，本 Skill 可以完成诊断与方案，但不能虚构执行结果。

## 输入

- 必需：业务目标、范围、对象、现状证据和已知约束。
- 建议：历史数据、流程或系统资料、目标值、owner、预算、SLA、合规要求。
- 可选：竞品、用户反馈、事故、实验、财务和上下游契约。

## 输出

- 适用法规和责任矩阵
- 证据缺口与风险等级
- 整改、监控和律师升级方案
- 决策记录、执行计划、指标验收、风险与未知项。

## 工作原理

1. 确认司法辖区、适用主体、交易或行为、时间点、目标和需要作出的业务决策
2. 收集合同、政策、证据、通信和当前官方规则，区分事实、主张、假设和法律问题
3. 建立权利义务、责任主体、期限、证据和潜在救济矩阵，按影响与可能性分级
4. 提出低风险可执行方案、替代文本、控制和保全动作，不把业务建议包装成确定法律结论
5. 识别必须升级当地律师、监管专家或诉讼代理人的触发条件和最晚时间
6. 输出审查记录、责任人、整改、审批、持续监控和特权/敏感信息边界

## 专业检查维度

- 司法辖区与经营主体
- 产品、进口和市场准入
- 消费者、平台和信息披露
- 海关、制裁和供应链证据
- 更新日期、主管机关和升级门槛

## 技能包组成

- `SKILL.md`：AI 执行入口与强制工作流。
- `references/professional-checklist.md`：专业检查表和失败模式。
- `assets/delivery-template.md`：可直接复用的交付模板。
- `examples/README.md`：调用示例。
- `eval/acceptance.md`：可判定验收标准。
- `README.md`：中文产品文档与迭代记录。

## 使用方式

1. 提供目标、范围、现状证据和限制条件。
2. 要求 AI 先读 `SKILL.md` 与 reference，再检查真实资料。
3. 让 AI 使用 asset 交付，并按 eval 自检；涉及写入或上线时另行确认权限。

## 验收标准

- 范围、术语、时间和数据口径明确。
- 关键结论均有证据、公式或清晰推理支撑。
- 方案覆盖正常路径、异常路径、上下游和责任边界。
- 动作有优先级、owner、依赖、指标、护栏和停止条件。
- 未知项、不确定性、风险和恢复方案被明确记录。
