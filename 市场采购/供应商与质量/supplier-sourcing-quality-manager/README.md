# 供应商寻源与质量管理器

> skill id：supplier-sourcing-quality-manager
> 当前版本：1.0.1
> 产品状态：可用
> 所属分类：市场采购 / 供应商与质量

## 产品定位

低价不是供应能力，样品合格也不等于量产受控。

## 解决的问题

- 供应商报价不可比且隐藏模具、包装和返工成本
- 样品不错但量产材料和工艺漂移
- 单一供应商或长交期放大缺货风险

## 适用场景

- 寻找并比较 3-5 家供应商
- 设计样品测试和验货规则
- MOQ、价格、交期和付款谈判准备

## 合规与权限

- 合规配置：`market-procurement`（采购与供应合规基线）。
- 执行前必须读取 `references/compliance-baseline.md`，完成司法辖区、数据、权限、授权、人工复核和审计检查。
- Skill 只能提供第一层规则约束，不能替代系统权限、真实授权、正式审批或专业法律意见。

## 不适用范围与边界

- 不得把供应商自述、证书截图或平台等级当作已验证事实
- 不得只按最低单价排序
- 未经样品和生产证据不得建议大货
- 规格、缺陷和验收必须可测量，避免好、优质、差不多等模糊词

## 输入

- 产品需求、目标市场与渠道
- 候选供应商报价和样品资料
- 质量、包装、MOQ、交期与付款底线

## 输出

- RFQ 与供应商对比表
- 样品测试与质量控制计划
- 商业和供应风险清单
- 试单/量产建议

## 工作原理

1. 确认产品规格、目标市场、销售渠道、预估量、质量底线、预算和交期
2. 将要求拆成 must-have、可协商和直接淘汰项，建立可测量规格
3. 向多个候选供应商发送一致 RFQ，收集阶梯价、MOQ、样品、模具、包装、认证、产能、交期和付款条件
4. 验证主体、工厂能力、关键工序、外包环节、质量体系和历史证据，不把平台徽章当充分证明
5. 设计盲比样品计划，记录材料、尺寸、功能、外观、气味、耐久、包装和运输测试
6. 建立黄金样、BOM、缺陷分级、AQL 或适用验收规则、检验方法和签样变更控制
7. 规划产前、首件、中期、尾期和装运前质量节点以及不合格处置
8. 比较总落地成本、现金条件、交期波动、沟通、可扩展性和单一来源风险
9. 输出首选、备选、条件性通过或淘汰结论以及试单和量产 gate

## 技能包组成

- SKILL.md：AI 执行入口。
- references/sourcing-quality-checklist.md：专业检查表。
- assets/supplier-scorecard.md：交付模板。
- eval/acceptance.md：可判定验收。
- skill.json：机器可读产品元数据。
- README.md：中文产品文档与版本记录。

## 使用方式

1. 提供真实业务输入和目标市场/渠道。
2. 让 AI 读取 SKILL.md 与指定 reference。
3. 使用 asset 模板交付并运行验收。

## 验收标准

- 所有供应商是否使用同一规格和报价口径
- 关键声明是否有可核验证据
- 样品、黄金样和量产变更是否受控
- 缺陷分级、检验和索赔责任是否明确
