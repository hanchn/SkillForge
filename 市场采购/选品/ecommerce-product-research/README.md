# 跨境电商选品研究

> skill id：ecommerce-product-research
> 当前版本：1.0.2
> 产品状态：可用
> 所属分类：市场采购 / 选品

## 产品定位

在采购和投放前，用需求、竞争、利润、供应链、合规和渠道适配证据判断一个产品机会是否值得继续。

## 解决的问题

- 选品只看热度和销量估算，忽略现金流与风险
- Amazon、Shopify 和 TikTok Shop 的渠道适配被混成一个总分
- 第三方销量、关键词和供应商声明容易被当成事实

## 适用场景

- 评估单个产品或批量比较候选产品
- 判断产品更适合 Amazon、独立站还是 TikTok Shop
- 建立低、中、高三档利润模型与低成本验证计划

## 合规与权限

- 合规配置：`market-procurement`（采购与供应合规基线）。
- 执行前必须读取 `references/compliance-baseline.md`，完成司法辖区、数据、权限、授权、人工复核和审计检查。
- Skill 只能提供第一层规则约束，不能替代系统权限、真实授权、正式审批或专业法律意见。

## 不适用范围与边界

- 不虚构销量、搜索量、费用、供应商报价、认证或限制
- 致命合规、IP 和现金流风险不能被综合高分抵消
- 没有足够证据时给出验证结论，不直接建议重仓

## 输入

- 候选产品或类目与目标市场
- 渠道选项、预算、利润底线和启动窗口
- 可选供应商、趋势、评论、市场和成本证据

## 输出

- 产品机会报告
- 情景利润模型
- 渠道匹配矩阵
- kill criteria 与分阶段验证计划

## 工作原理

1. 定义客户任务和当前替代方案
2. 建立带日期和局限的证据日志
3. 分析需求质量、竞争结构和差评机会
4. 计算完整贡献利润与敏感性
5. 筛查供应链、合规、IP 和现金周期
6. 按渠道评分并给出继续、验证、搁置或拒绝结论

## 深度执行标准

- 覆盖完整业务理解、主场景与相邻变体、上下游、方案取舍、工具失败、异常、交付、验收和复盘。
- 复杂任务先读 `references/scenario-playbook.md`；存在决策或审批时使用 `assets/decision-record-template.md`。
- 不以篇幅代替深度，每项判断必须关联真实对象、证据、责任和结果。

## 技能包组成

- references/research-framework.md：证据、致命筛查和经济模型
- assets/opportunity-scorecard.md：机会评分卡
- eval/acceptance.md：研究质量验收

通用入口文件：

- SKILL.md：AI 执行入口与专业工作流
- skill.json：机器可读元数据
- INVOCATION.md：标准调用顺序与失败处理

## 使用方式

1. 将整个 ecommerce-product-research/ 目录交给支持 skill 的 AI 或复制到对应技能目录。
2. 让 AI 先读取 SKILL.md；README.md 用于人类理解产品能力和升级历史。
3. 按任务需要提供输入证据，执行后使用 eval/acceptance.md 验收。
4. 若修改能力、边界、输入输出或资源，应同步更新 README.md 的当前说明和版本记录。

## 验收标准

- 市场、预算、利润底线和时间窗口明确
- 经济模型覆盖广告、退货、履约和库存风险
- 渠道适配分开判断
- 结论包含证据置信度和停止条件

## 版本与升级规则

- 当前能力以本 README 和 skill.json 的版本为准。
- 每次能力、工作流、资源、输入输出或边界发生实质变化，必须提升版本并补充下方记录。
- 仅修正文案错字或链接时可不提升版本，但仍应保证产品说明与实际文件一致。
- 历史记录只写已发生变化，不把计划中的能力写成已经交付。
