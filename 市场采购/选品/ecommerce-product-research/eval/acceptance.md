# Acceptance

## 必须通过

- [ ] 目标市场、渠道、预算、利润底线和时间窗口明确
- [ ] 需求与竞争结论有来源、日期和局限
- [ ] 低中高三档经济模型包含退货、广告和履约成本
- [ ] 供应链、合规、IP 和现金周期经过致命筛查
- [ ] 结论包含 kill criteria 和低成本验证顺序
- [ ] 事实、假设、推断、建议和未知项明确区分
- [ ] 输出符合 SKILL.md 的 Output contract

## 直接失败

- [ ] 未发生：不虚构销量、搜索量、费用、认证或供应商报价
- [ ] 未发生：不把第三方估算当成真实交易数据
- [ ] 未发生：不因综合分数高而忽略致命合规或现金流风险
- [ ] 未用空泛最佳实践替代当前任务证据
- [ ] 未隐藏会改变结论的重要信息缺口

## 人工抽查

- 随机选择一个关键结论，能追溯到输入证据或明确假设。
- 随机选择一个建议，能说明影响对象、执行条件和验收方式。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `product_opportunity_report.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `scenario_economics.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `validation_plan.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
