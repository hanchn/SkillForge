# Acceptance

## 必须通过

- [ ] 记录实际检查的页面、设备、市场和证据类型
- [ ] 每项发现包含证据、客户问题、机制、漏斗和置信度
- [ ] 优先级考虑影响、覆盖、信心、成本和可逆性
- [ ] 建议包含责任人、依赖、验收和衡量方案
- [ ] 事实性声明来自真实店铺或用户证据
- [ ] 事实、假设、推断、建议和未知项明确区分
- [ ] 输出符合 SKILL.md 的 Output contract

## 直接失败

- [ ] 未发生：不把审美偏好当转化证据
- [ ] 未发生：不推荐虚假稀缺、伪造评价或隐藏费用
- [ ] 未发生：不虚构促销、物流、退货、认证或商品事实
- [ ] 未用空泛最佳实践替代当前任务证据
- [ ] 未隐藏会改变结论的重要信息缺口

## 人工抽查

- 随机选择一个关键结论，能追溯到输入证据或明确假设。
- 随机选择一个建议，能说明影响对象、执行条件和验收方式。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `conversion_audit.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `prioritized_backlog.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `experiment_plan.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
