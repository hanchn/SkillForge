# 独立站产品上下架管理器

> skill id：storefront-product-publishing-manager
> 当前版本：1.0.2
> 产品状态：可用
> 所属分类：渠道运营 / 多渠道运营 / 独立站

## 产品定位

用于安全规划、执行和验证独立站产品上架、定时发布、临时隐藏、永久退市、归档和删除。它把产品状态变更视为涉及商品、库存、履约、SEO、广告、数据和客服的协同操作，而不是简单切换 active 状态。

## 解决的问题

- 上架时遗漏库存、价格、履约、声明、市场或渠道配置。
- 下架后广告、外链、组合商品、订阅和搜索结果继续指向失效页面。
- 草稿、隐藏、缺货、归档和删除被混为一谈，造成不可逆损失。
- 后台显示成功，但客户页面、购物车、feed 或分析系统并未真正生效。

## 适用场景

- 新产品立即或定时上架。
- 指定市场、语言或销售渠道发布。
- 临时停售并保留恢复路径。
- 产品永久退市、替代、归档或删除。
- 审查一批商品是否具备上下架条件。
- Shopify 或其他自有独立站的产品生命周期管理。

## 合规与权限

- 合规配置：`channel-commerce`（渠道经营合规基线）。
- 执行前必须读取 `references/compliance-baseline.md`，完成司法辖区、数据、权限、授权、人工复核和审计检查。
- Skill 只能提供第一层规则约束，不能替代系统权限、真实授权、正式审批或专业法律意见。

## 不适用范围与边界

- 没有稳定产品标识时不执行真实变更。
- 没有明确授权时不修改线上状态、重定向、广告或 feed。
- 不从商品图片或旧文案推断价格、库存、合规、配送和产品声明。
- 不把删除作为默认下架方式。
- 没有可写的认证工具时，只生成审批和执行方案，不假装已经操作成功。

## 输入

- 店铺、环境、产品和变体标识。
- 上架、定时、隐藏、退市、归档或删除目标。
- 影响市场、语言、渠道、生效时间与时区。
- 商品事实、库存、履约、合规和授权证据。
- 可选广告、feed、SEO、组合、订阅和支持依赖。

## 输出

- 当前状态快照。
- 阻塞项与警告清单。
- 上下架依赖和有序执行计划。
- URL、SEO 与替代商品决策。
- 授权、执行、验证和回滚记录。
- 监控窗口与剩余工作。

## 工作原理

1. 先确认具体商品、环境和动作类别。
2. 保存变更前状态并扫描商品、库存、履约、营销与 SEO 依赖。
3. 选择满足目标的最小可逆状态变化。
4. 在真实变更前取得明确授权。
5. 按顺序执行并从客户、运营和数据表面验证。
6. 监控异常并保留恢复方案与审计记录。

## 深度执行标准

- 覆盖完整业务理解、主场景与相邻变体、上下游、方案取舍、工具失败、异常、交付、验收和复盘。
- 复杂任务先读 `references/scenario-playbook.md`；存在决策或审批时使用 `assets/decision-record-template.md`。
- 不以篇幅代替深度，每项判断必须关联真实对象、证据、责任和结果。

## 技能包组成

- SKILL.md：AI 执行入口与安全流程。
- references/lifecycle-checklist.md：上下架前置检查和阻塞条件。
- assets/publishing-change-plan.md：审批、执行与审计模板。
- eval/acceptance.md：身份、授权、依赖、验证和回滚验收。
- skill.json：机器可读产品元数据。
- README.md：中文产品文档和版本记录。

## 使用方式

1. 将整个 skill 文件夹交给支持的平台。
2. 让 AI 读取 SKILL.md 并确认当前是否有可写的认证工具。
3. 提供唯一商品标识、目标动作、影响范围、生效时间和授权人。
4. 执行后用 eval/acceptance.md 检查客户页面、渠道、订单和回滚状态。

## 验收标准

- 商品、变体、环境、市场、渠道和动作没有歧义。
- 所有阻塞项在执行前关闭或明确停止。
- 线上变更有授权和 before/after 证据。
- SEO、广告、feed、组合、订阅和订单依赖已处理。
- 客户侧和运营侧验证通过，失败时能按计划恢复。

## 版本与升级规则

- 实质能力升级必须同步 skill.json 和本 README。
- 新增平台专用执行器时，需要记录平台、权限、可逆性和测试结果。
- 版本记录只写已经交付的能力。
