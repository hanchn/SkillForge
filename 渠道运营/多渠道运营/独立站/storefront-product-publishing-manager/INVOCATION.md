# Invocation

1. 读取 skill.json、SKILL.md 和 references/lifecycle-checklist.md。
2. 确认店铺、环境、商品、变体、动作、范围、生效时间和授权。
3. 保存当前状态并用 assets/publishing-change-plan.md 记录依赖与变更顺序。
4. 若缺少授权或可写工具，只交付计划，不执行或假报成功。
5. 执行后验证客户页面、市场渠道、库存履约、购物车结账、SEO、feed、活动和分析。
6. 使用 eval/acceptance.md 验收并返回回滚路径。

## 失败处理

- 商品身份或目标动作不唯一：停止变更并要求稳定标识。
- 商品事实、库存、履约或合规阻塞：保留草稿状态并列出责任人。
- 依赖无法同步：缩小变更范围或调整执行窗口。
- 验证失败：按已批准的回滚方案恢复，不扩大修改范围。
