# Acceptance

- [ ] 店铺、环境、商品、变体、市场、渠道和动作唯一明确。
- [ ] before 状态和依赖证据已保存。
- [ ] 价格、库存、履约、声明、URL 和渠道配置经过核验。
- [ ] 广告、feed、组合、订阅、订单、支持和外链影响已检查。
- [ ] 真实线上变更具有明确授权。
- [ ] 选择了满足目标的最小可逆状态，而非默认删除。
- [ ] 客户页面、搜索、购物车、结账和运营后台验证完成。
- [ ] SEO 和重定向决策与用户价值相关，不统一跳首页。
- [ ] 失败时存在可执行回滚步骤和责任人。
- [ ] 没有可写工具时，结果明确标记为计划而不是已执行。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `publishing_change_plan.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `preflight_and_dependency_report.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `execution_and_verification_record.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
