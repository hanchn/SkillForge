# Amazon 市场 Rank 与实时趋势调研方法

## 先区分 Rank 类型

- BSR：ASIN 在指定 Amazon 类目中的 Best Sellers Rank，不等于某个关键词的自然搜索位置。
- 自然搜索位：在明确站点、查询词、时间和观察条件下出现的非广告位置。
- Sponsored 位置：广告展示位置，必须与自然位分开记录。
- 榜单位置：Best Sellers、Movers & Shakers、New Releases 等榜单中的位置和榜单类型。
- 业务侧自定义 Rank：第三方工具或内部模型计算结果，必须记录算法、来源和版本。

## 当前数据采集

每条观察至少记录：站点、类目、查询词、ASIN、Rank 类型、位置、页面或数据源、采集时间、设备/地区条件、是否登录和限制。

可用来源按任务权限选择：

1. 当前 Amazon 商品页、搜索结果和类目榜单。
2. 使用者授权提供的 Seller Central、Brand Analytics、Search Query Performance、广告搜索词等数据。
3. Amazon 搜索建议及当前竞品 PDP。
4. 用户已购买或授权的第三方关键词、Rank 和历史趋势工具。
5. 其他外部趋势来源只能作为补充，不能冒充 Amazon 站内搜索量或 Rank。

## 竞品样本

- 先按产品类型、使用场景、目标人群、价格带和关键属性确认直接相关性。
- Rank 高但产品任务不同的 ASIN 只能作为邻近参考，不能直接决定关键词与文案。
- 同时保留高 Rank、增长信号、新品和与贵司定位相近的样本，避免只学习头部成熟 Listing。

## 趋势判断

- “实时”只能描述本次采集时点的当前状态。
- 7/30/90 天趋势需要连续快照或含历史序列的数据源。
- 单次 Rank 变化不能解释为需求、转化或算法变化；同时检查价格、广告、促销、评论、库存、季节和内容变化。
- 报告趋势方向、波动、样本缺口和置信度，不输出伪精确因果结论。

## 刷新规则

- 每次正式生成或重大改版前刷新当前快照。
- 大促、季节变化、类目迁移、价格带变化或关键词策略调整时重新采集。
- 在研究报告中写明数据截止时间和下一次建议刷新时间。
