# Amazon PDP SEO 与 GEO 关键词方法

## 关键词来源

- 使用者提供并确认优先策略的关键词库。
- 已验证商品名称、属性、材质、规格、使用场景、问题和受众语言。
- 当前 Amazon 搜索建议、相关查询、竞品 PDP 和类目表达。
- 使用者授权提供的 Brand Analytics、Search Query Performance、广告搜索词和转化数据。
- 可靠的外部趋势或第三方工具，只按其真实口径使用。

每个候选词记录：原词、标准词、语言、来源、采集日期、查询意图、商品相关性、事实证据、趋势、竞争代理、用户优先模式、目标字段、采用/排除和理由。

## SEO 字段映射

- 标题：核心品类、关键区分属性和重要适用语境，优先清晰自然，不堆砌同义词。
- 五点：把高价值搜索意图与可证明的利益、属性、使用方式和限制结合。
- 描述/A+ 输入：补足场景、实体关系、使用步骤、比较边界和长尾语义，不重复标题与五点。
- Backend Search Terms：放置仍相关但不宜前台重复的同义词和变体；遵守当前站点、类目和字段规则。
- 变体：父子体共享核心语义，颜色、尺寸、套装等差异必须与真实子体一致。

## GEO 语义覆盖

GEO 在本 Skill 中指让检索、推荐和生成式回答系统更容易正确理解商品，不代表保证被引用或提高 Amazon 排名。

- 明确商品实体、品类、关键属性、适用对象、使用场景、兼容关系和限制。
- 用自然语言回答用户真实问题：它是什么、适合谁、何时使用、如何使用、与什么不同、有哪些限制。
- 每个答案必须来自商品事实台账；没有证据的问题保留为空或转补证。
- 避免空泛形容词、机械关键词重复、伪造比较、评价、销量、健康效果或用户体验。

## 优先词处理

- `REQUIRED_CANDIDATE` 代表必须逐词评估，不代表必须写入。
- `PREFERRED` 只在相关性、事实证据和字段适配相当时优先。
- `REFERENCE_ONLY` 不提高布局优先级。
- 所有优先词仍受相关性、商品事实、竞品商标、禁用词、字段限制和自然表达约束。

## 完成标准

- 每个重要词能追溯来源和选择理由。
- 重要查询意图被一个最合适字段覆盖，而不是全字段重复。
- 已采用、排除、待验证和用户优先词分别列明。
- SEO 与 GEO 建议都不承诺排名、索引、推荐或 AI 引用结果。
