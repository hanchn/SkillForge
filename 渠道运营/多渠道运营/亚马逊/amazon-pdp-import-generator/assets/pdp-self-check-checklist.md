# Amazon PDP 草稿上线前自检

## A. 生成前确认

- [ ] 已询问是否有关键词库，并记录版本、站点、语言、来源、更新时间和 owner。
- [ ] 已单独询问是否优先参考给定关键词，并记录优先模式。
- [ ] 已锁定 Marketplace、类目、语言、产品类型、变体主题和当前模板版本。

## B. 商品事实

- [ ] 标题、五点、描述、Search Terms 和属性字段全部可追溯到商品事实台账。
- [ ] 材质、尺寸、包装、兼容、认证、性能、安全、产地、保修和适用人群无图片推断。
- [ ] 父子体共享事实一致，真实变体差异没有被覆盖。

## C. Rank 与趋势

- [ ] BSR、自然搜索位、广告位和趋势榜位置分开记录。
- [ ] 当前数据有站点、类目/查询、ASIN、时间和来源。
- [ ] 竞品在产品、场景、人群和价格带上相关。
- [ ] 单次快照没有被描述成 7/30/90 天趋势。

## D. SEO

- [ ] 每个采用关键词都有来源、相关性、意图和目标字段。
- [ ] 用户优先词仍通过事实、相关性、商标、禁词和字段规则检查。
- [ ] 标题自然可读，五点不重复堆词，描述补充语义，Backend Terms 去重。
- [ ] 没有竞品品牌、无关高流量词、主观最高级或未证明声明。

## E. GEO

- [ ] 商品实体、属性、适用对象、使用场景、兼容关系和限制表达清楚。
- [ ] 关键用户问题有简洁、事实支持的答案或明确标记缺口。
- [ ] 未承诺搜索排名、索引、推荐、Amazon Rufus/其他 AI 引用结果。

## F. 字段与内容质量

- [ ] 使用当前站点和类目规则检查字符/字节、必填字段、禁用内容和模板映射。
- [ ] 拼写、语法、单位、数字、标点、语言、本地化和移动端可读性通过。
- [ ] 图片主附图、SKU、父子体、变体主题和内容中的颜色/尺寸一致。
- [ ] Search Terms 等隐藏字段没有机械重复前台内容。

## G. 交付状态

- [ ] 所有失败项已修复并重新检查，或记录阻塞、owner 和截止时间。
- [ ] 未通过 Seller Central/current template 验证时，输出只称为 review draft。
- [ ] 市场研究、关键词地图、解析日志、自检和版本记录随草稿交付。
