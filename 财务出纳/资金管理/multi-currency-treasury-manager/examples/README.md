- 调用目标：使用 `multi-currency-treasury-manager` 完成该场景的完整业务结果，不只给原则。
- 必须提供：目标、范围、真实现状证据、owner、时间、权限和已知约束。
- 需要追问：现有数据/系统/模板/词库/规则、使用者优先级、不可改变项、下游接收方式和验收人。
- 预期交付：币种资金头寸、汇率暴露与情景、换汇、归集、权限和预警方案。
