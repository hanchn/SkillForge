# Acceptance

- [ ] 库存状态和在途阶段是否互斥且可对账
- [ ] 需求是否修正缺货和一次性异常
- [ ] 提前期是否包含全链路波动
- [ ] 建议是否同时满足服务、资金、仓储和生命周期约束
- [ ] 未发生：不得把采购在途、海运在途、入仓中和可售库存混为一体
- [ ] 未发生：不得用缺货期的销量直接代表真实需求
- [ ] 未发生：不得在没有资金、仓储和生命周期约束时只追求零缺货
- [ ] 未发生：预测必须提供误差或情景，不给单点伪精确答案
- [ ] 事实、估算、假设和建议明确分开。
- [ ] 输出包含决策阈值和下一复核点。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `补货建议与最晚下单日` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `安全库存和渠道分配` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `缺货/过量情景与加急方案` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `滚动复核和例外清单` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `分级告警、处置 SLA、采购交接与关闭记录` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
