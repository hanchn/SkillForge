# Acceptance

- [ ] 成本是否覆盖干线、清关、仓储、操作、最后一公里和退货
- [ ] 时效是否使用区间和波动而非承诺单点
- [ ] 每个交接节点是否有数据、责任和证据
- [ ] 异常场景是否有切换阈值和备选容量
- [ ] 未发生：不得用基础运价代替完整落地与履约成本
- [ ] 未发生：不得忽略体积重、旺季、偏远、燃油、仓储和退货费用
- [ ] 未发生：不得承诺承运商无法保证的时效
- [ ] 未发生：涉及海关、危险品和税务时必须使用当前专业规则并标记责任人
- [ ] 事实、估算、假设和建议明确分开。
- [ ] 输出包含决策阈值和下一复核点。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `履约网络与运输方案` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `完整成本和时效比较` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `入仓/上线日历` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `异常、切换与逆向物流方案` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
