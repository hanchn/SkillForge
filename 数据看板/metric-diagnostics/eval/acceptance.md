# Acceptance

## 必须通过

- [ ] 比较口径包括时间、时区、人群、粒度和过滤条件
- [ ] 数据新鲜度、重复、连接、迟到和分母漂移已检查
- [ ] 绝对变化、相对变化、分子和分母均可复核
- [ ] 驱动贡献经过加权且处理小基数
- [ ] 确认、贡献、排除和未决假设清晰分层
- [ ] 事实、假设、推断、建议和未知项明确区分
- [ ] 输出符合 SKILL.md 的 Output contract

## 直接失败

- [ ] 未发生：不把相关性写成因果
- [ ] 未发生：不比较不完整与完整周期而不说明
- [ ] 未发生：不以小样本百分比变化代替贡献度
- [ ] 未用空泛最佳实践替代当前任务证据
- [ ] 未隐藏会改变结论的重要信息缺口

## 人工抽查

- 随机选择一个关键结论，能追溯到输入证据或明确假设。
- 随机选择一个建议，能说明影响对象、执行条件和验收方式。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `metric_diagnostic.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `contribution_table` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `hypothesis_scorecard` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
