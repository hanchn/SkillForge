# 用户生命周期、会员与挽留经理验收

## 场景与输入

- [ ] 已选择主场景，并检查是否叠加其他场景或高风险变体
- [ ] 已询问使用者现有数据、系统、模板、规则、词库、优先级和不可改变项
- [ ] 使用了真实输入并标明证据来源、日期、版本、范围和可信度
- [ ] 定义了对象、术语、指标、上下游、责任边界与不做事项

## 专业维度

- [ ] `客户身份、同意、新老客和会员口径` 已有事实、判断、未知项或不适用理由
- [ ] `cohort、RFM、LTV、复购周期和成熟度` 已有事实、判断、未知项或不适用理由
- [ ] `激活、首购、二购、活跃、沉默、流失和召回状态` 已有事实、判断、未知项或不适用理由
- [ ] `积分、等级、权益、推荐、成本和负债` 已有事实、判断、未知项或不适用理由
- [ ] `触达频控、优惠依赖、对照实验、增量留存和贡献利润` 已有事实、判断、未知项或不适用理由

## 交付物

- [ ] `新客、老客、会员和流失定义及数据质量报告` 已完整交付，并能由下游直接使用或审批
- [ ] `生命周期分层、状态机、触发、频控和跨渠道旅程` 已完整交付，并能由下游直接使用或审批
- [ ] `会员权益、积分负债、成本、毛利和体验模型` 已完整交付，并能由下游直接使用或审批
- [ ] `召回挽留实验、增量评估、止损与复盘` 已完整交付，并能由下游直接使用或审批

## 决策与异常

- [ ] 推荐方案与维持现状、低风险备选进行了适用条件、成本、收益、风险和可逆性比较
- [ ] 覆盖缺数据、口径冲突、权限不足、依赖失败、低置信度、超时和部分成功
- [ ] 动作具有 owner、审批人、依赖、截止、验收、护栏、停止条件与恢复路径
- [ ] 工具或脚本执行结果经过业务验证，没有把调用成功写成业务完成

## 真实性与复盘

- [ ] 区分事实、计算、推断、建议、未知项和使用者偏好
- [ ] 未虚构实施、验证、效果、权限、实时数据或专业结论
- [ ] 已定义领先指标、结果指标、护栏指标、复盘时间和版本维护人
