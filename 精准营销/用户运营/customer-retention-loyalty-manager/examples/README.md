# 用户生命周期、会员与挽留经理场景示例

## 场景 1：新客首购后培育与二购提升

- 调用目标：使用 `customer-retention-loyalty-manager` 完成该场景的完整业务结果，不只给原则。
- 必须提供：目标、范围、真实现状证据、owner、时间、权限和已知约束。
- 需要追问：现有数据/系统/模板/词库/规则、使用者优先级、不可改变项、下游接收方式和验收人。
- 预期交付：新客、老客、会员和流失定义及数据质量报告、生命周期分层、状态机、触发、频控和跨渠道旅程、会员权益、积分负债、成本、毛利和体验模型、召回挽留实验、增量评估、止损与复盘。
- 故障注入：至少测试缺数据、口径冲突、权限不足或依赖失败中的一种，并给出暂停、补证、转人工或恢复路径。
