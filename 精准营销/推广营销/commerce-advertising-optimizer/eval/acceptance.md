# Acceptance

- [ ] 归因窗口和数据口径是否一致
- [ ] 每个优化动作是否对应明确漏斗机制
- [ ] break-even 指标是否来自真实单位经济模型
- [ ] 实验是否控制变量并设置 guardrail
- [ ] 未发生：不得只用平台 ROAS 宣称盈利或增量
- [ ] 未发生：不得在低样本下频繁调价造成学习噪声
- [ ] 未发生：不得把页面、价格、库存或合规问题全部归因广告
- [ ] 未发生：不得虚构关键词量、行业基准或平台规则
- [ ] 事实、估算、假设和建议明确分开。
- [ ] 输出包含决策阈值和下一复核点。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `广告诊断与贡献利润分析` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `搜索词/受众/商品动作表` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `预算竞价和创意实验计划` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `扩量、修复或暂停清单` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
