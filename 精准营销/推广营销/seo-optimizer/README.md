# SEO 内容优化师

> skill id：seo-optimizer
> 当前版本：2.0.0
> 产品状态：可用
> 所属分类：跨境运营 / 推广营销 / SEO

## 产品定位

围绕真实搜索意图、页面业务任务和可验证事实，规划、撰写或审计 SEO 内容。

## 解决的问题

- 只堆关键词而没有真正完成用户搜索任务
- 商品事实、关键词数据和竞品结论容易被编造
- 内容、技术 SEO、站点架构和转化问题混在一起，难以执行

## 适用场景

- 新建商品页、集合页、落地页或文章的 SEO brief
- 优化标题、H1、meta、正文结构和内链
- 审查页面意图匹配、内容缺口、关键词蚕食和技术阻塞

## 不适用范围与边界

- 不虚构搜索量、难度、排名、竞品流量或商品事实
- 不保证排名
- 不使用关键词堆砌、门页、隐藏文本或无关 schema

## 输入

- 页面、内容或 URL
- 目标市场、语言、受众和转化目标
- 真实业务事实与可选关键词、SERP、流量数据

## 输出

- SEO 审计或优化内容
- 搜索意图与查询地图
- 页面结构、metadata 和内链方案
- 技术阻塞与衡量计划

## 工作原理

1. 确认页面类型、市场和业务任务
2. 建立事实来源表并区分实测关键词与假设词
3. 分析搜索意图和可用 SERP 证据
4. 设计信息结构并生成内容、metadata、内链和 schema 建议
5. 检查技术前提并定义业务衡量方法

## 技能包组成

- SKILL.md：SEO 核心工作流
- references/content-audit.md：审计检查表
- assets/seo-content-brief.md：内容简报模板
- eval/acceptance.md：事实、意图和技术验收

通用入口文件：

- SKILL.md：AI 执行入口与专业工作流
- skill.json：机器可读元数据
- INVOCATION.md：标准调用顺序与失败处理
- BASE_PROMPT.md：跨平台执行原则
- agents/openai.yaml：支持平台的界面元数据，如该文件存在
- README.md：当前中文产品文档与版本记录

## 使用方式

1. 将整个 seo-optimizer/ 目录交给支持 skill 的 AI 或复制到对应技能目录。
2. 让 AI 先读取 SKILL.md；README.md 用于人类理解产品能力和升级历史。
3. 按任务需要提供输入证据，执行后使用 eval/acceptance.md 验收。
4. 若修改能力、边界、输入输出或资源，应同步更新 README.md 的当前说明和版本记录。

## 验收标准

- 关键词数据与假设明确区分
- 页面只有清晰的主要搜索任务
- 产品和权威声明可以追溯
- 技术阻塞与内容建议分开
- 衡量不只依赖排名

## 版本与升级规则

- 当前能力以本 README 和 skill.json 的版本为准。
- 每次能力、工作流、资源、输入输出或边界发生实质变化，必须提升版本并补充下方记录。
- 仅修正文案错字或链接时可不提升版本，但仍应保证产品说明与实际文件一致。
- 历史记录只写已发生变化，不把计划中的能力写成已经交付。
