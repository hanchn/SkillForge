# GEO 机器可读内容发布专家场景示例

## 场景 1：静态资料或 PDP/PIM/CMS 接口接入与可信知识库建设

- 调用目标：使用 `geo-machine-readable-publisher` 完成该场景的完整业务结果，不只给原则。
- 必须提供：目标、范围、真实现状证据、owner、时间、权限和已知约束。
- 需要追问：现有数据/系统/模板/词库/规则、使用者优先级、不可改变项、下游接收方式和验收人。
- 预期交付：数据源、API、字段权限与可信度清单、带来源、版本、有效期和冲突状态的 GEO 可信知识库、llms.txt、llms-full.txt 和 JSON-LD 生成包、批量清单、自检报告、发布和回滚记录。
- 故障注入：至少测试缺数据、口径冲突、权限不足或依赖失败中的一种，并给出暂停、补证、转人工或恢复路径。
