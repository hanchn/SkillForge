# Acceptance

## 必须通过

- [ ] 触发范围与相邻 skill 可区分。
- [ ] 输入事实、假设、未知项和授权边界明确。
- [ ] 工作流包含领域特有判断，不是泛化的收集、执行、返回三步。
- [ ] 每个关键结论可追溯到证据或明确假设。
- [ ] 输出合同规定具体文件、章节、表格或状态。
- [ ] 声明的脚本和资源真实存在，路径与 skill.json 一致。

## 直接失败

- [ ] 未出现占位文本、虚构数据、不存在的脚本或空泛验收。
- [ ] 未把高风险专业结论包装成无条件确定事实。
- [ ] 未用 README 代替 SKILL.md 作为 AI 入口。

## 人工复核

- 用一个正常案例和一个输入不足案例分别试运行。
- 检查技能是否在缺证据时正确降级，而不是补造完整答案。
