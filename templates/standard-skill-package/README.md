- 明确与相邻 skill 的分工。

## 工作原理

1. 用中文概述 SKILL.md 中的核心判断链。
2. 说明证据、规则、工具和人工复核如何配合。
3. 不在这里复制整份 AI 指令。

## 技能包组成

- SKILL.md：AI 执行入口
- skill.json：机器可读产品元数据
- README.md：中文产品文档与版本记录
- INVOCATION.md：调用和失败处理
- references/：专业参考资料，如存在
- assets/：输出模板或配置，如存在
- scripts/：确定性执行脚本，如存在
- eval/acceptance.md：验收标准

## 使用方式

1. 将整个 skill 文件夹交给支持的平台。
2. 让 AI 优先读取 SKILL.md。
3. 提供任务证据并在交付后运行验收。

## 验收标准

- 写出可客观判断的通过条件。
- 写出会直接导致交付失败的情况。

## 版本与升级规则

- 实质能力升级必须同步 skill.json 版本和本 README。
- 版本记录只写已经交付的变化，不写计划能力。
- 文案修正可不提升版本，但必须保持文档与实际文件一致。
