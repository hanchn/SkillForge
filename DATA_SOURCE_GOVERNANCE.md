# SkillForge 数据与合规接口治理

## 目标

多数企业 Skill 的价值来自真实数据、当前系统状态和可验证证据，而不是 LLM 自身知识。模型负责澄清、组织、计算、诊断、生成草案和形成决策闭环，不能替代销量、市场、库存、财务、候选人、法规、代码或平台事实源。

## 全库强制流程

1. 先向使用者确认 CSV、Excel、JSON、Parquet、日志、文档等静态文件，系统导出、数据库视图/查询结果、官方 API、内部接口、已授权第三方 API和允许查询的公开来源。
2. 使用各 Skill 的 `assets/data-source-intake-template.md` 登记 owner、授权目的、字段、口径、截止、刷新、质量、敏感等级和公开范围。
3. 使用 `references/data-source-and-api-gate.md` 判断 `DATA_READY`、`STATIC_SNAPSHOT`、`RESEARCH_ONLY`、`REVIEW_REQUIRED` 或 `BLOCKED`。
4. 只有 `DATA_READY` 才能形成依赖实时事实的正式结论；静态快照必须标有效期，其他状态必须保留数据缺口和补数计划。
5. 每个输出都要可追溯到来源、查询/文件版本、采集时间和转换口径，并指定下一次刷新 owner。

## 合规接口不是“能调通的接口”

接口还必须满足提供方与合同允许、业务目的明确、权限最小、字段必要、地域和主体适用、个人/敏感数据受控、下游发布范围获批、凭证安全、限流成本可控、删除撤回可传播、日志可审计。

不得在对话、Skill、脚本参数、代码、日志或交付物中粘贴生产 token、key、cookie、密码或私钥。凭证通过环境变量、密钥管理或已授权连接器提供。

## 数据不足时的边界

- 不用模型记忆生成实时销量、搜索量、Rank、趋势、价格、库存、汇率、薪酬、法规或平台规则。
- 不把第三方估算、公开抓取、平台后台、公司订单和财务账混成同一真实性等级。
- 不把单次快照写成趋势，不把相关性写成因果，不把平台归因写成公司增量利润。
- 无数据时可以交付问题清单、数据字典、接口需求、采集方案、分析框架或明确假设的草案，但不能伪装成已验证业务结论。

## 静态表质量门禁

CSV、Excel 或其他静态表必须记录来源系统、导出人/owner、导出时间、数据截止、编码、sheet/列定义、主键、粒度、单位、币种、时区和适用范围；检查缺失、重复、异常值、截断、状态口径和跨表对账。静态文件只能代表对应快照，除非包含可靠历史序列，否则不能推导趋势。

## 选品特别门禁

选品至少要判断市场需求、关键词与趋势、竞品和价格带、Rank 与评价、渠道费用与广告、退货和季节、供应商报价/MOQ/交期/质量、物流关税、合规/IP、贡献利润、现金周期和库存风险。缺少改变投资结论的关键数据时，只能给出验证顺序与 kill criteria，不能推荐直接采购或放量。
