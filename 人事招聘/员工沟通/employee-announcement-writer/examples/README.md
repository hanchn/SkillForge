# 企业员工公告撰写专家场景示例

## 场景 1：入职、放假、培训、福利和办公通知

- 调用目标：使用 `employee-announcement-writer` 完成该场景的完整业务结果，不只给原则。
- 必须提供：目标、范围、真实现状证据、owner、时间、权限和已知约束。
- 需要追问：现有数据/系统/模板/词库/规则、使用者优先级、不可改变项、下游接收方式和验收人。
- 预期交付：正式公告正文、标题、摘要和多渠道短版、审批、发布、问答和反馈清单。
- 故障注入：至少测试缺数据、口径冲突、权限不足或依赖失败中的一种，并给出暂停、补证、转人工或恢复路径。

## 反例

- 只复述最佳实践，没有读取真实证据。
- 没有询问使用者已有资料、优先级和限制，就直接生成定稿。
- 只输出一张表或一段文案，没有上下游、异常、验收和责任闭环。
