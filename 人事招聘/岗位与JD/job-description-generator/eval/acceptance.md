# 招聘需求自动生成验收

- [ ] `可编辑候选人 JD` 已完整生成并可由发起人修改
- [ ] `HR 精准岗位培训文档` 已完整生成并支持零沟通交接
- [ ] `内部岗位说明书` 已与 JD 和筛选标准保持一致
- [ ] 创建需求后一次生成候选人 JD、HR 培训、内部岗位说明和有效 JSON 筛选契约
- [ ] 正常路径没有要求发起人与 HR 再次沟通交接
- [ ] 四份产物共享 request ID、package version 和 rubric version
- [ ] HR 培训覆盖岗位原因、日常工作、上下游、工具术语、职级差异和常见误解
- [ ] 每个筛选标准有稳定 ID、类别、岗位相关理由、召回词、直接/替代证据和最低证据
- [ ] 关键词只用于召回，敏感属性和无关代理变量已禁用
- [ ] 文案修改不会无意义重置 rubric
- [ ] 实质要求变化会同步全部产物、递增 rubric 并标记旧结果过期
- [ ] 非关键未知项保留为待确认但不阻塞草案
- [ ] 关键冲突进入 `REVIEW_REQUIRED` 并只列最少修复字段
- [ ] 无阻断项输出 `READY_FOR_HR`，简历筛选 Skill 可直接消费
- [ ] 可选市场/薪酬调研没有阻塞已批准的完整招聘需求
