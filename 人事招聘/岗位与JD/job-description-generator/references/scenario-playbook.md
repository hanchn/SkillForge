# 招聘需求自动生成场景

## 场景 1：创建招聘需求

直接生成完整招聘包和筛选契约。非关键未知项保留为待确认；无冲突时输出 `READY_FOR_HR`。

## 场景 2：发起人修改自动生成内容

- 文案修改：只更新候选人 JD。
- 岗位要求修改：同步更新培训、内部说明和筛选规则，递增 rubric 版本。
- 地点、用工、待遇或合规条件修改：暂停自动流转并标记需审核内容。

## 场景 3：现有 JD 迁移

从现有 JD 生成培训文档和筛选契约；无法证明岗位相关性的硬门槛降级为待确认，不静默进入自动筛选。

## 场景 4：按需市场与薪酬调研

仅在明确请求或政策要求时执行；当前样本和公司批准口径分开，调研失败不阻塞已经批准且信息完整的招聘需求。

## 完成门槛

- 四份核心产物共享同一 request ID 和版本。
- HR 无需再次解释岗位即可发布、筛选和准备面试。
- 简历筛选 Skill 无需重读自然语言 JD 即可运行。
- 异常有明确原因、影响字段和修正动作，不把开会沟通当默认流程。
