# 无人值守筛选与预约验收

- [ ] `岗位证据候选池` 已完整生成且每项结论可追溯到 criterion ID 和简历证据
- [ ] `筛选漏斗和异常归因` 已完整生成并通过确定性阈值脚本验证
- [ ] `面试预约、会议室、面试官和候选人确认状态` 已按平台 ID 对账
- [ ] 有效 `screening-spec.json` 通过校验且未被重新解释
- [ ] 清单默认不包含简历全文，符号链接不会读取目录外文件
- [ ] 重复文件有不同 Candidate ID 并能指向首个内容记录
- [ ] 关键词只用于召回，候选池结论包含岗位证据
- [ ] 输出完整筛选漏斗、最大淘汰标准和异常归因
- [ ] 1000 份唯一可解析简历少于 10 人时触发默认低产出预警
- [ ] 阈值可按岗位覆盖，不被描述为永久统一标准
- [ ] 正常候选人按幂等键创建预约、会议室、面试官通知和候选人确认邮件
- [ ] 候选人确认后才写 `INTERVIEW_CONFIRMED`
- [ ] 邮件失败不会重复创建会议；超时会释放临时资源
- [ ] 筛选与预约异常均生成去重的飞书事件，不暴露简历正文或敏感信息
- [ ] API 请求成功没有被误写成候选人已确认或问题已关闭
- [ ] 自动化只形成候选池和预约，最终录用/永久拒绝保留授权决定
