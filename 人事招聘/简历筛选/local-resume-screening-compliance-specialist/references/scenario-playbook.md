# 无人值守筛选与预约场景

## 场景 1：正常自动化批次

有效筛选契约和连接器就绪时，自动解析、生成候选池、预约面试、通知面试官并等待候选人二次确认。无异常输出 `AUTOMATION_COMPLETED`。

## 场景 2：候选池过低

触发低产出阈值后，按解析率、岗位族相关率、最大淘汰标准、证据不足率和规则版本依次归因；通过飞书发送包含具体修复动作的聚合告警。

## 场景 3：预约部分失败

会议已创建但邮件失败时，不重复创建会议；按 booking/event ID 对账并只补发通知。状态无法确认时停止自动重试、保留占位状态并发飞书 P1 告警。

## 场景 4：候选人未确认或改期

超时释放临时会议室和日历占位，写回招聘平台并发送飞书提醒；候选人选择改期时沿用同一 application 记录但使用新的幂等键。

## 场景 5：规则变化

JD 实质修改导致 rubric 版本变化时停止旧批次，标记旧结果过期并按新规则重跑。不得合并不同 rubric 版本的漏斗。
