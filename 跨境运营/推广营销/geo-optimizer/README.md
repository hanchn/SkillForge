# GEO 引用就绪优化师

> skill id：geo-optimizer  
> 当前版本：2.0.0  
> 产品状态：可用  
> 所属分类：跨境运营 / 推广营销 / GEO

## 产品定位

通过实体一致性、声明溯源和可独立抽取的答案模块，提高生成式答案引擎准确理解与引用内容的可能性。

## 解决的问题

- 品牌和商品声明缺乏来源、日期和适用范围
- 内容只有营销口号，模型难以准确抽取
- GEO 容易滑向伪造权威、垃圾外链和社区操纵

## 适用场景

- 审计官网或帮助中心的 AI 引用就绪程度
- 建立品牌、产品或研究内容的 claim ledger
- 将页面改成带证据、条件和限制的答案模块

## 不适用范围与边界

- 不虚构统计、专家、评论、认证、研究或第三方背书
- 不保证被任何模型收录或引用
- 不推荐伪造共识、社区灌水、付费链接伪装或百科操纵

## 输入

- 品牌、产品、页面或内容
- 目标市场与受众问题
- 声明来源、canonical URL 和可选模型观察

## 输出

- 实体与问题地图
- 声明台账
- 引用就绪差距与答案模块
- 技术发布、合规分发和衡量方案

## 工作原理

1. 确认实体、市场和 canonical 来源
2. 为每项重要声明记录来源、日期、范围和置信度
3. 统一实体名称与产品身份
4. 将真实问题写成直接答案、证据、条件和限制
5. 检查结构化数据、索引和公开资料一致性
6. 设计抽样式 AI 答案准确度衡量

## 技能包组成

- references/citation-readiness.md：引用就绪标准
- assets/claim-ledger.md：声明台账模板
- eval/acceptance.md：事实与分发伦理验收

通用入口文件：

- SKILL.md：AI 执行入口与专业工作流
- skill.json：机器可读元数据
- INVOCATION.md：标准调用顺序与失败处理
- BASE_PROMPT.md：跨平台执行原则
- agents/openai.yaml：支持平台的界面元数据，如该文件存在
- README.md：当前中文产品文档与版本记录

## 使用方式

1. 将整个 geo-optimizer/ 目录交给支持 skill 的 AI 或复制到对应技能目录。
2. 让 AI 先读取 SKILL.md；README.md 用于人类理解产品能力和升级历史。
3. 按任务需要提供输入证据，执行后使用 eval/acceptance.md 验收。
4. 若修改能力、边界、输入输出或资源，应同步更新 README.md 的当前说明和版本记录。

## 验收标准

- 所有重要声明被支持、收窄、补证或移除
- 答案模块脱离上下文仍能正确理解
- 高风险声明保留日期、地域、样本和限制
- 没有引用保证或操纵式分发建议

## 版本与升级规则

- 当前能力以本 README 和 skill.json 的版本为准。
- 每次能力、工作流、资源、输入输出或边界发生实质变化，必须提升版本并补充下方记录。
- 仅修正文案错字或链接时可不提升版本，但仍应保证产品说明与实际文件一致。
- 历史记录只写已发生变化，不把计划中的能力写成已经交付。
