# 电商履约物流规划师

> skill id：ecommerce-fulfillment-logistics-planner
> 当前版本：1.0.0
> 产品状态：可用
> 所属分类：跨境运营 / 物流 / 跨境履约

## 产品定位

物流方案必须同时满足成本、时效、可追踪、合规、容量和异常恢复。

## 解决的问题

- 只比较头程单价，忽略全链路费用和入仓波动
- Amazon 与独立站订单缺少清晰履约路由
- 延误、扣关、丢损和退货没有责任与恢复方案

## 适用场景

- 首批试单和补货运输方式选择
- FBA、FBM、3PL 和直发组合设计
- 旺季、爆仓或承运商中断应急

## 不适用范围与边界

- 不得用基础运价代替完整落地与履约成本
- 不得忽略体积重、旺季、偏远、燃油、仓储和退货费用
- 不得承诺承运商无法保证的时效
- 涉及海关、危险品和税务时必须使用当前专业规则并标记责任人

## 输入

- 产品与包装尺寸重量、起运和目的地
- 渠道需求、订单结构与服务承诺
- 承运商、仓库、FBA/3PL 报价和限制

## 输出

- 履约网络与运输方案
- 完整成本和时效比较
- 入仓/上线日历
- 异常、切换与逆向物流方案

## 工作原理

1. 确认产品尺寸重量、危险或特殊属性、起运地、目的市场、渠道、销量、承诺时效和退货要求
2. 建立端到端节点图：工厂、质检、提货、出口、干线、清关、仓库、FBA/3PL、最后一公里和逆向物流
3. 统一比较计费重、体积重、固定费、操作费、关税税费、仓储、退货、丢损和加急成本
4. 用低中高时效和波动评估海运、空运、快递、铁路或本地备货组合
5. 设计 FBA、FBM、3PL 和独立站订单路由、库存缓冲、分单、缺货和峰值容量
6. 定义标签、箱规、预约、ASN、追踪、POD、清关文件和数据交接责任
7. 建立延误、扣关、拒收、丢件、破损、地址错误、仓库爆仓和承运商中断 playbook
8. 输出首选、备选、验证批次、时间表、成本区间、SLA 和切换阈值
9. 用准时率、交付时长、成本/单、损坏、丢件、退款、首次投递和仓库差错复盘

## 技能包组成

- skill.json：机器可读产品元数据。
- README.md：中文产品文档与版本记录。

## 使用方式

1. 提供真实业务输入和目标市场/渠道。
2. 让 AI 读取 SKILL.md 与指定 reference。
3. 使用 asset 模板交付并运行验收。

## 验收标准

- 成本是否覆盖干线、清关、仓储、操作、最后一公里和退货
- 时效是否使用区间和波动而非承诺单点
- 每个交接节点是否有数据、责任和证据
- 异常场景是否有切换阈值和备选容量
