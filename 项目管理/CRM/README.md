# CRM 项目

## 分类定位

客户关系系统的产品架构和后续细分 Skill。

## 选择原则

- 先用上层通用 Skill 定义边界、口径和总体方案，再用语言、框架、渠道或系统 Skill 深化。
- 一个任务可以组合多个 Skill，但必须指定主 Skill，避免重复 ownership。
- README 是中文产品文档；执行时先读对应 Skill 的 `SKILL.md`。

## 当前 Skill

- [CRM 产品架构师](crm-product-architecture/README.md)：围绕统一客户、同意偏好、互动、服务、分群和生命周期运营设计 CRM 产品架构。 Use when an AI needs to handle 跨渠道客户中心, 营销与客服 CRM, 客户 360 和生命周期运营; produce CRM 能力地图, 客户身份与同意模型, 互动、分群、任务和集成方案; and apply evidence, explicit boundaries, validation, and rollback instead of generic advice.
