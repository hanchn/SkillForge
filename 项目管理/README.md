## 定位

项目管理目录用于按公司内部项目或系统组织项目资料、里程碑、负责人和导航。产品架构师属于互联网研发产品职能，相关 Skill 已统一迁至 `互联网研发/产品/系统产品架构/`；本目录不再直接存放产品架构 Skill。

## 目录规则

- 项目代号目录是浏览层，不因为有 README.md 就成为 skill。
- 项目管理类 Skill 才放在 `项目管理/<项目代号>/<skill-id>/`；产品架构 Skill 放在 `互联网研发/产品/系统产品架构/<系统>/`。
- skill-id 使用稳定英文小写连字符命名。
- 一个能力只服务某个项目时，放在该项目目录。
- 同一能力被多个项目复用且业务规则一致时，放入 公共/。
- 名称相同但规则、状态机或数据口径不同的能力，不强行抽成公共 skill。
- 每个正式 skill 必须有 SKILL.md、skill.json、中文产品 README、版本记录和可判定验收。

## 项目说明维护要求

每个项目 README 后续应持续补充：

1. 公司内部正式名称与项目定位。
2. 业务范围、非范围和目标用户。
3. 上下游系统与关键集成关系。
4. 核心业务对象、状态、规则和数据口径。
5. 已有 skill、规划 skill 和版本演进。
6. 负责人、资料来源和敏感信息边界。
