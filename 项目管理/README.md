# 项目管理

> 目录类型：业务浏览层  
> 当前状态：项目骨架已建立  
> 维护对象：公司内部业务系统及其细分 skill

## 定位

项目管理目录用于按公司内部项目或系统沉淀 skill。IMS、TMS、OMS、OFS、WMS 目前只作为项目代号保留；具体中文名称、业务边界和上下游关系以后按公司真实口径补充，不在仓库中自行推断。

## 目录规则

- 项目代号目录是浏览层，不因为有 README.md 就成为 skill。
- 具体 skill 放在 项目管理/<项目代号>/<skill-id>/。
- skill-id 使用稳定英文小写连字符命名。
- 一个能力只服务某个项目时，放在该项目目录。
- 同一能力被多个项目复用且业务规则一致时，放入 公共/。
- 名称相同但规则、状态机或数据口径不同的能力，不强行抽成公共 skill。
- 每个正式 skill 必须有 SKILL.md、skill.json、中文产品 README、版本记录和可判定验收。

## 项目说明维护要求

每个项目 README 后续应持续补充：

1. 公司内部正式名称与项目定位。
2. 业务范围、非范围和目标用户。
3. 上下游系统与关键集成关系。
4. 核心业务对象、状态、规则和数据口径。
5. 已有 skill、规划 skill 和版本演进。
6. 负责人、资料来源和敏感信息边界。
