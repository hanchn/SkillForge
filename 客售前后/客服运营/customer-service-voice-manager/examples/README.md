# 跨境客售前后运营专家场景示例

## 场景 1：邮件、聊天和平台消息的售前售中售后接待

- 调用目标：使用 `customer-service-voice-manager` 完成该场景的完整业务结果，不只给原则。
- 必须提供：目标、范围、真实现状证据、owner、时间、权限和已知约束。
- 需要追问：现有数据/系统/模板/词库/规则、使用者优先级、不可改变项、下游接收方式和验收人。
- 预期交付：客售前后答复与处理方案、客服 SOP、FAQ、话术、权限和升级矩阵、咨询、售后、质检、培训与 VOC 改进报告。
- 故障注入：至少测试缺数据、口径冲突、权限不足或依赖失败中的一种，并给出暂停、补证、转人工或恢复路径。
