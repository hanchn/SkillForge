# 基础工具 Skill 地图

基础工具只承载跨部门可复用能力。业务角色专属决策、指标和周报继续留在各业务目录，避免通用模板覆盖专业口径。

## 消息与告警

- `message-alert-orchestration-specialist`：把业务与系统事件统一为分级、去重、抑制、升级、回执、关闭和复盘流程，适配飞书、钉钉、企业微信及经确认的企业 QQ/内部网关。

## 汇报与复盘

- `general-company-weekly-report`：用于没有专属角色 Skill 的团队或跨部门通用周报；已有角色周报时优先使用角色 Skill。
- `retrospective-action-closure-facilitator`：用于事故、项目、活动和经营复盘，追踪行动到效果验证而非任务打勾。

## 组合方式

1. 业务 Skill 产生可监控事件后交给消息告警 Skill，不让每个业务 Skill 自建平台发送逻辑。
2. 周报发现连续阻塞或重复风险时调用复盘 Skill。
3. 复盘行动达到阈值、逾期或复发时可再次生成告警事件。

所有工具都必须先读取数据与合规接口门禁；真实发送、发布或个人信息处理需要对应 owner 授权。
