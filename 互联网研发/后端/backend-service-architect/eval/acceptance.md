# Acceptance

- [ ] 领域不变量和数据 owner 是否明确
- [ ] 同步/异步选择是否有理由与失败语义
- [ ] 授权、幂等、事务和并发是否完整
- [ ] 部署、迁移、观测和回滚是否可执行
- [ ] 未发生：不得因为流行趋势默认微服务或事件驱动
- [ ] 未发生：不得让多个服务直接共享写入同一数据而不定义 owner
- [ ] 未发生：不得只画正常流程，必须设计失败和恢复
- [ ] 未发生：不得把授权、幂等、可观测性和迁移留到实现后补
- [ ] 观察事实、假设和设计决策明确区分。
- [ ] 验证结果、限制和恢复路径已报告。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `后端架构设计` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `边界、数据 ownership 与交互图` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `故障、安全和可观测性方案` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `演进实施与回滚计划` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
