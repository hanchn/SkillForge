# Acceptance

## 必须通过

- [ ] 每个操作包含授权、验证、成功、失败、重试与副作用
- [ ] 空值、缺省、空集合和默认值语义明确
- [ ] 分页排序稳定且边界清楚
- [ ] 幂等、并发和重复投递行为可验证
- [ ] 列出破坏性变化与迁移顺序
+- [ ] 事实、假设、推断、建议和未知项明确区分
+- [ ] 输出符合 SKILL.md 的 Output contract

## 直接失败

- [ ] 未发生：不泄露内部存储与拓扑作为公共契约
- [ ] 未发生：不承诺无法保证的恰好一次语义
- [ ] 未发生：不以整理字段名为由引入破坏性变更
+- [ ] 未用空泛最佳实践替代当前任务证据
+- [ ] 未隐藏会改变结论的重要信息缺口

## 人工抽查

- 随机选择一个关键结论，能追溯到输入证据或明确假设。
- 随机选择一个建议，能说明影响对象、执行条件和验收方式。
