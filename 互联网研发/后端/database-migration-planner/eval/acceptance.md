# Acceptance

- [ ] 生产规模、锁和运行时是否经过评估
- [ ] expand/migrate/contract 是否允许新旧版本共存
- [ ] backfill 是否可重入、可节流和可对账
- [ ] 恢复方案是否处理数据语义而不只回退代码
- [ ] 未发生：不得把向下迁移脚本等同于可恢复数据
- [ ] 未发生：不得在未知表规模和锁行为下直接执行生产 DDL
- [ ] 未发生：不得删除旧列、表或索引直到依赖和保留期验证完成
- [ ] 未发生：不得依赖不可重入的一次性 backfill
- [ ] 观察事实、假设和设计决策明确区分。
- [ ] 验证结果、限制和恢复路径已报告。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `分阶段迁移计划` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `DDL/backfill 与兼容发布顺序` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `对账、监控和 gate` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `回滚/前滚恢复与清理计划` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
