# Acceptance

- [ ] 影响、严重度、授权和事件角色是否明确
- [ ] 证据是否在变更前保存并统一时间线
- [ ] 假设是否有预期信号和反证
- [ ] 恢复是否从用户、系统和数据三个层面验证
- [ ] 未发生：不得在事故中执行无恢复点的大范围破坏性命令
- [ ] 未发生：不得把时间相关性直接写成根因
- [ ] 未发生：不得泄露凭据、个人信息或敏感日志
- [ ] 未发生：没有授权时只提供诊断与变更建议，不擅自扩大生产操作
- [ ] 观察事实、假设和设计决策明确区分。
- [ ] 验证结果、限制和恢复路径已报告。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `事件状态与影响` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `时间线和假设记分卡` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `止损/恢复动作与验证` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `根因、数据修复和后续改进` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
