# SvelteKit 前端架构师

> skill id：sveltekit-frontend-architecture  
> 当前版本：1.0.0  
> 产品状态：可用  
> 所属分类：互联网研发 / 前端 / 框架 / SvelteKit

## 产品定位

围绕文件路由、load、form actions、服务端边界和 adapter 设计生产级 SvelteKit 应用。

## 解决的问题

- 工作依赖个人经验，输入、过程和交付口径不稳定。
- 只关注局部动作，缺少边界、上下游、异常、数据和经营结果闭环。
- 建议无法追溯证据，也没有负责人、验收、止损或演进机制。

## 适用场景

- SvelteKit 新项目
- SSR/SSG/SPA 混合架构
- SvelteKit 数据与部署治理

## 不适用范围与边界

- 不得把服务端私密数据引入客户端 bundle，也不得忽略 adapter 的运行环境约束。
- 关键口径或权限缺失时，本 Skill 可以完成诊断与方案，但不能虚构执行结果。

## 输入

- 必需：业务目标、范围、对象、现状证据和已知约束。
- 建议：历史数据、流程或系统资料、目标值、owner、预算、SLA、合规要求。
- 可选：竞品、用户反馈、事故、实验、财务和上下游契约。

## 输出

- SvelteKit 路由和渲染架构
- 数据、表单与状态边界
- adapter、观测和部署方案
- 决策记录、执行计划、指标验收、风险与未知项。

## 工作原理

1. 检查现有仓库、运行拓扑、质量目标、团队边界和约束；区分已验证事实、假设与未知项
2. 建立系统上下文、关键用户旅程或请求路径，明确边界外依赖和非功能目标
3. 按业务能力划分模块，定义 ownership、依赖方向、公共契约和禁止跨越的边界
4. 设计数据、状态、错误、权限、缓存、并发和资源生命周期，不只覆盖正常路径
5. 评估备选架构的复杂度、性能、可靠性、交付成本和可逆性，记录决策理由
6. 制定增量实施、兼容迁移、验证、观测、灰度和回滚计划

## 专业检查维度

- route、layout 与项目结构
- load、form actions 和 remote functions
- server-only 模块与 secrets
- SSR、prerender 和客户端状态
- hooks、errors、adapter 和 observability

## 技能包组成

- `SKILL.md`：AI 执行入口与强制工作流。
- `references/professional-checklist.md`：专业检查表和失败模式。
- `assets/delivery-template.md`：可直接复用的交付模板。
- `examples/README.md`：调用示例。
- `eval/acceptance.md`：可判定验收标准。
- `README.md`：中文产品文档与迭代记录。

## 使用方式

1. 提供目标、范围、现状证据和限制条件。
2. 要求 AI 先读 `SKILL.md` 与 reference，再检查真实资料。
3. 让 AI 使用 asset 交付，并按 eval 自检；涉及写入或上线时另行确认权限。

## 验收标准

- 范围、术语、时间和数据口径明确。
- 关键结论均有证据、公式或清晰推理支撑。
- 方案覆盖正常路径、异常路径、上下游和责任边界。
- 动作有优先级、owner、依赖、指标、护栏和停止条件。
- 未知项、不确定性、风险和恢复方案被明确记录。
