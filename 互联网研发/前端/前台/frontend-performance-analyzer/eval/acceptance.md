# Acceptance

- [ ] 现场与实验室数据是否分开并能解释差异
- [ ] trace 是否把用户症状关联到具体资源或主线程任务
- [ ] 修复是否有同条件 before/after 和业务 guardrail
- [ ] 预算是否覆盖关键页面、资源和交互而非只有总分
- [ ] 未发生：不得只用单次 Lighthouse 或本机高速网络结果下结论
- [ ] 未发生：不得通过延迟功能、隐藏内容或取消必要验证制造指标改善
- [ ] 未发生：不得把服务端 TTFB、第三方和客户端执行问题混成一个前端结论
- [ ] 未发生：不得声称优化有效而没有同条件 before/after 证据
- [ ] 每项问题都有可复现证据、影响和复测方式。
