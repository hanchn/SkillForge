# Web 性能与无障碍审计师

> skill id：web-performance-accessibility-auditor
> 当前版本：1.0.4
> 产品状态：可用
> 所属分类：互联网研发 / 前端 / 前台 / 质量审计

## 产品定位

先测量具体页面与用户旅程，再修关键路径；性能分数和自动扫描都不是最终用户体验。

## 解决的问题

- 只优化实验室分数而真实用户仍然卡顿
- 鼠标可用但键盘和读屏无法完成任务
- 修复图片或脚本后引入布局、交互和业务回归

## 适用场景

- 审计商品页或业务流程的 Core Web Vitals
- 检查表单、弹层和动态内容无障碍
- 发布前建立性能预算和回归门槛

## 合规与权限

- 合规配置：`product-engineering`（产研合规基线）。
- 执行前必须读取 `references/compliance-baseline.md`，完成司法辖区、数据、权限、授权、人工复核和审计检查。
- Skill 只能提供第一层规则约束，不能替代系统权限、真实授权、正式审批或专业法律意见。

## 不适用范围与边界

- 不得只凭 Lighthouse 单次分数下结论
- 不得用 aria 修补本可用原生语义解决的问题
- 不得通过隐藏内容、延迟交互或删除功能制造虚假性能提升
- 不得声称自动扫描已覆盖全部无障碍问题

## 输入

- URL、构建或页面代码
- 目标设备、网络和关键旅程
- 可选 RUM、性能 trace、无障碍报告和用户反馈

## 输出

- 性能与无障碍审计
- 根因和优先级 backlog
- 修复验收与复测结果
- 预算和监控建议

## 工作原理

1. 确定页面、设备、网络、用户旅程、登录状态、地区、浏览器和发布版本
2. 收集现场数据与实验室证据，分开记录 LCP、INP、CLS、资源瀑布、主线程和交互症状
3. 定位关键渲染路径、最大内容元素、输入响应、布局移动、字体、图片、脚本和第三方影响
4. 用键盘走完整流程，检查语义结构、地标、标题、表单标签、错误、焦点、弹层和动态提示
5. 检查对比度、缩放、响应式、触控目标、横竖屏、reduced motion 和高对比模式
6. 运行自动工具但手工验证关键问题，避免把无告警当合格
7. 将发现写成证据、用户影响、根因、修复、风险、owner 和验收方式
8. 按用户影响、覆盖、置信度、成本和回归风险排序
9. 实施或建议最小修复，并在同一条件下复测
10. 建立预算、监控和回归测试，防止一次性优化回退

## 深度执行标准

- 覆盖完整业务理解、主场景与相邻变体、上下游、方案取舍、工具失败、异常、交付、验收和复盘。
- 复杂任务先读 `references/scenario-playbook.md`；存在决策或审批时使用 `assets/decision-record-template.md`。
- 不以篇幅代替深度，每项判断必须关联真实对象、证据、责任和结果。

## 技能包组成

- SKILL.md：AI 执行入口。
- references/audit-checklist.md：专业检查表。
- assets/web-quality-report-template.md：交付模板。
- eval/acceptance.md：可判定验收。
- README.md：中文产品文档与版本记录。

## 使用方式

1. 提供目标、现有代码或生产证据与约束。
2. 让 AI 先读 SKILL.md 和指定 reference。
3. 按仓库实际命令验证，使用 asset 完成交付。

## 验收标准

- 实验室和现场数据是否分开
- 关键用户旅程是否用键盘和辅助语义验证
- 每项发现是否包含可复现证据和根因
- 修复后是否在相同环境复测并检查业务回归
