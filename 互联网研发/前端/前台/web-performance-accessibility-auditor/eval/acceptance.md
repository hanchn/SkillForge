# Acceptance

- [ ] 实验室和现场数据是否分开
- [ ] 关键用户旅程是否用键盘和辅助语义验证
- [ ] 每项发现是否包含可复现证据和根因
- [ ] 修复后是否在相同环境复测并检查业务回归
- [ ] 未发生：不得只凭 Lighthouse 单次分数下结论
- [ ] 未发生：不得用 aria 修补本可用原生语义解决的问题
- [ ] 未发生：不得通过隐藏内容、延迟交互或删除功能制造虚假性能提升
- [ ] 未发生：不得声称自动扫描已覆盖全部无障碍问题
- [ ] 观察事实、假设和设计决策明确区分。
- [ ] 验证结果、限制和恢复路径已报告。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `性能与无障碍审计` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `根因和优先级 backlog` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `修复验收与复测结果` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `预算和监控建议` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
