# Acceptance

- [ ] locale、市场、货币和时区是否分开建模
- [ ] 所有 key、变量、复数和富文本标签是否匹配
- [ ] 长文本、伪本地化和 RTL 是否验证
- [ ] URL、metadata、canonical 和 hreflang 是否一致
- [ ] 目标语言语义是否安排合格复核人
- [ ] 未发生：不得把机器翻译通顺度当成完整本地化验收
- [ ] 未发生：不得通过字符串拼接构造需要语法变化的句子
- [ ] 未发生：不得把国家、语言、货币和时区视为同一个概念
- [ ] 未发生：不熟悉目标语言时必须标记语言质量需母语或专业复核
- [ ] 每项问题都有可复现证据、影响和复测方式。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `多语言覆盖与 key 漂移报告` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `locale 行为和格式矩阵` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `布局/RTL/SEO 问题清单` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `复测与专业语言复核清单` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
