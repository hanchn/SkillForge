# Acceptance

- [ ] 是否读取并复用现有框架、组件和命令
- [ ] 所有用户可见状态是否覆盖
- [ ] 键盘、焦点、标签和错误提示是否可用
- [ ] 类型、测试、构建和视觉检查是否通过
- [ ] 未发生：不得忽略仓库现有模式重新搭一套架构
- [ ] 未发生：不得只实现截图正常态而遗漏交互和失败状态
- [ ] 未发生：不得为了视觉相似牺牲语义、键盘或响应式
- [ ] 未发生：不得伪造后端契约、真实数据或未提供的产品决策
- [ ] 观察事实、假设和设计决策明确区分。
- [ ] 验证结果、限制和恢复路径已报告。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `可运行前端实现` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `测试与验证结果` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `行为状态和限制说明` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
