# 前端功能实现工程师

> skill id：frontend-feature-implementer
> 当前版本：1.0.1
> 产品状态：可用
> 所属分类：互联网研发 / 前端 / 前台 / 功能实现

## 产品定位

先理解现有产品和代码约束，再实现完整交互状态，不只拼出静态正常态。

## 解决的问题

- 页面看起来完成但加载、错误和空状态不可用
- 新代码绕过设计系统和仓库模式
- 桌面截图还原后移动端和键盘体验失败

## 适用场景

- 实现商品页、表单、仪表盘或工作流
- 根据 Figma/截图还原现有应用页面
- 补齐现有功能的响应式、错误和权限状态

## 合规与权限

- 合规配置：`product-engineering`（产研合规基线）。
- 执行前必须读取 `references/compliance-baseline.md`，完成司法辖区、数据、权限、授权、人工复核和审计检查。
- Skill 只能提供第一层规则约束，不能替代系统权限、真实授权、正式审批或专业法律意见。

## 不适用范围与边界

- 不得忽略仓库现有模式重新搭一套架构
- 不得只实现截图正常态而遗漏交互和失败状态
- 不得为了视觉相似牺牲语义、键盘或响应式
- 不得伪造后端契约、真实数据或未提供的产品决策

## 输入

- 需求、设计、截图或目标行为
- 现有前端仓库和项目规范
- 可选 API 契约、测试账号和验收条件

## 输出

- 可运行前端实现
- 测试与验证结果
- 行为状态和限制说明

## 工作原理

1. 读取仓库说明、框架、路由、设计系统、状态方案、测试命令和相邻实现
2. 把需求拆成用户任务、页面结构、交互、权限、数据依赖、状态和验收标准
3. 复用现有 token、组件、布局和数据访问方式，记录必须新增的抽象
4. 先实现语义结构和数据边界，再补视觉、响应式和动效
5. 覆盖初始、加载、成功、空、校验失败、权限失败、网络失败、重试和禁用状态
6. 实现键盘操作、焦点顺序、标签、错误关联、可读状态提示和必要的 reduced-motion 行为
7. 处理并发提交、重复点击、过期响应、离开页面和回退导航
8. 编写与风险匹配的组件、交互和集成测试
9. 运行格式、类型、测试和构建，并在可用时做桌面/移动视觉核对
10. 报告修改文件、行为、验证、已知限制和待补证据

## 技能包组成

- SKILL.md：AI 执行入口。
- references/implementation-checklist.md：专业检查表。
- assets/feature-handoff-template.md：交付模板。
- eval/acceptance.md：可判定验收。
- README.md：中文产品文档与版本记录。

## 使用方式

1. 提供目标、现有代码或生产证据与约束。
2. 让 AI 先读 SKILL.md 和指定 reference。
3. 按仓库实际命令验证，使用 asset 完成交付。

## 验收标准

- 是否读取并复用现有框架、组件和命令
- 所有用户可见状态是否覆盖
- 键盘、焦点、标签和错误提示是否可用
- 类型、测试、构建和视觉检查是否通过
