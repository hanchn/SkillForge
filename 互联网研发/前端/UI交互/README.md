# UI 交互

## 分类定位

设计页面组件状态、操作路径、反馈、动效、可访问性以及可供前端实现测试的交互规范。

## 选择原则

- 先用上层通用 Skill 定义边界、口径和总体方案，再用语言、框架、渠道或系统 Skill 深化。
- 一个任务可以组合多个 Skill，但必须指定主 Skill，避免重复 ownership。
- README 是中文产品文档；执行时先读对应 Skill 的 `SKILL.md`。

## 当前 Skill

- [UI 交互设计与实现专家](ui-interaction-designer/README.md)：把业务任务和界面结构转化为状态完整、反馈明确、可访问、可实现且可测试的 UI 交互规范。适用于新页面或组件交互设计、复杂表单、列表、弹窗和工作流改版、交互缺失、反馈不清或前端验收争议，交付用户流程与交互说明、组件状态、事件和反馈矩阵、动效、可访问性、埋点和验收清单。
