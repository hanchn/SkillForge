# UI 交互设计与实现专家场景示例

## 场景 1：新页面或组件交互设计

- 调用目标：使用 `ui-interaction-designer` 完成该场景的完整业务结果，不只给原则。
- 必须提供：目标、范围、真实现状证据、owner、时间、权限和已知约束。
- 需要追问：现有数据/系统/模板/词库/规则、使用者优先级、不可改变项、下游接收方式和验收人。
- 预期交付：用户流程与交互说明、组件状态、事件和反馈矩阵、动效、可访问性、埋点和验收清单。
- 故障注入：至少测试缺数据、口径冲突、权限不足或依赖失败中的一种，并给出暂停、补证、转人工或恢复路径。

## 反例

- 只复述最佳实践，没有读取真实证据。
