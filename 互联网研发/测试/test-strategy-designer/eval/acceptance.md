# Acceptance

## 必须通过

- [ ] 风险有可解释的概率、影响、可检测性和爆炸半径
- [ ] 每个高风险项映射到测试层和可信 oracle
- [ ] 环境、身份、数据、时钟与清理规则明确
- [ ] 发布门槛包含证据来源、责任人和停止条件
- [ ] 生产验证与回滚信号覆盖预发无法复现的风险
- [ ] 事实、假设、推断、建议和未知项明确区分
- [ ] 输出符合 SKILL.md 的 Output contract

## 直接失败

- [ ] 未发生：不按测试数量衡量质量
- [ ] 未发生：不默认测试环境等同生产
- [ ] 未发生：不为同一风险在每一层机械复制用例
- [ ] 未用空泛最佳实践替代当前任务证据
- [ ] 未隐藏会改变结论的重要信息缺口

## 人工抽查

- 随机选择一个关键结论，能追溯到输入证据或明确假设。
- 随机选择一个建议，能说明影响对象、执行条件和验收方式。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `test_strategy.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `risk_coverage_matrix.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `release_gates.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
