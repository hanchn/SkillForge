# Acceptance

## 必须通过

- [ ] 问题陈述不是功能复述，且证据与假设分离
- [ ] 每项需求有稳定编号并可独立评审
- [ ] 审批、取消、失败、重试等状态转换完整
- [ ] 验收标准可客观判定通过或失败
- [ ] 问题、需求、验收与指标能够追溯
- [ ] 事实、假设、推断、建议和未知项明确区分
- [ ] 输出符合 SKILL.md 的 Output contract

## 直接失败

- [ ] 未发生：不代替产品负责人做未授权的商业决策
- [ ] 未发生：不把假设包装成已确认事实
- [ ] 未发生：不把技术方案细节混成产品需求
- [ ] 未用空泛最佳实践替代当前任务证据
- [ ] 未隐藏会改变结论的重要信息缺口

## 人工抽查

- 随机选择一个关键结论，能追溯到输入证据或明确假设。
- 随机选择一个建议，能说明影响对象、执行条件和验收方式。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `requirements_package.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `traceability_matrix.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `open_decisions.md` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
