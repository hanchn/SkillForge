# Acceptance

## 必须通过

- [ ] 问题陈述不是功能复述，且证据与假设分离
- [ ] 每项需求有稳定编号并可独立评审
- [ ] 审批、取消、失败、重试等状态转换完整
- [ ] 验收标准可客观判定通过或失败
- [ ] 问题、需求、验收与指标能够追溯
+- [ ] 事实、假设、推断、建议和未知项明确区分
+- [ ] 输出符合 SKILL.md 的 Output contract

## 直接失败

- [ ] 未发生：不代替产品负责人做未授权的商业决策
- [ ] 未发生：不把假设包装成已确认事实
- [ ] 未发生：不把技术方案细节混成产品需求
+- [ ] 未用空泛最佳实践替代当前任务证据
+- [ ] 未隐藏会改变结论的重要信息缺口

## 人工抽查

- 随机选择一个关键结论，能追溯到输入证据或明确假设。
- 随机选择一个建议，能说明影响对象、执行条件和验收方式。
