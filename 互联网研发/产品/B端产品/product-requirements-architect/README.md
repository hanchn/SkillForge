# 产品需求架构师

把模糊需求变成决策完整、工程可实现、测试可验收的需求包。

## 适用场景

- 把这段口头需求整理成可评审 PRD，并标出真正阻塞开发的决策。
- 审查这份 PRD，找出无法测试、状态缺失和权限不清的部分。
- 为审批工作流补齐角色、状态、异常、验收标准和指标。

## 交付物

- requirements_package.md
- traceability_matrix.md
- open_decisions.md

## 边界

- 不代替产品负责人做未授权的商业决策
- 不把假设包装成已确认事实
- 不把技术方案细节混成产品需求

## 使用入口

1. 先读 SKILL.md。
2. 按 SKILL.md 指示读取 references 或 assets 中的专用资料。
3. 按 INVOCATION.md 执行并用 eval/acceptance.md 验收。

整个目录可作为独立跨平台 skill 包分发。README 只负责给人说明，不是执行入口。
