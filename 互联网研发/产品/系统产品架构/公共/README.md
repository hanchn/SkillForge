# 跨系统产品边界

## 分类定位

沉淀跨系统能力分工、主数据归属、交接契约和端到端异常闭环。

## 选择原则

- 先用上层通用 Skill 定义边界、口径和总体方案，再用语言、框架、渠道或系统 Skill 深化。
- 一个任务可以组合多个 Skill，但必须指定主 Skill，避免重复 ownership。
- README 是中文产品文档；执行时先读对应 Skill 的 `SKILL.md`。

## 当前 Skill

- [跨系统产品架构师](commerce-systems-product-architecture/README.md)：为贵司 OMS、IMS、OFS、CMS、WMS、TMS、CRM、PLM 建立业务能力分工、主数据归属、系统接口和端到端协作边界。适用于跨系统业务流程设计、系统职责重叠与空白治理、新系统接入和系统边界调整，交付业务能力与系统责任地图、主数据、状态与 source of truth 矩阵、跨系统流程、交接契约与异常闭环。
