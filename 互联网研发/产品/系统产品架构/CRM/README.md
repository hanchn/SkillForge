# CRM 产品架构

## 分类定位

客户关系系统的产品架构和后续细分 Skill。

## 选择原则

- 先用上层通用 Skill 定义边界、口径和总体方案，再用语言、框架、渠道或系统 Skill 深化。
- 一个任务可以组合多个 Skill，但必须指定主 Skill，避免重复 ownership。
- README 是中文产品文档；执行时先读对应 Skill 的 `SKILL.md`。

## 当前 Skill

- [CRM 产品架构师](crm-product-architecture/README.md)：围绕统一客户、身份解析、同意偏好、线索商机、互动服务、分群旅程和生命周期运营设计可审计的 CRM 产品架构。适用于跨渠道客户中心与客户 360、营销、销售与客售前后 CRM、线索、商机、服务工单和生命周期运营、CRM 重构、迁移、主数据与权限治理，交付CRM 业务能力与角色权限地图、客户身份、账户、联系人、同意和主数据模型、线索、商机、互动、工单、分群和旅程状态机、CRM 与 CDP/OMS/CMS/客服/营销/数据平台集成及迁移方案。
