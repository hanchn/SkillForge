# 全栈测试周报生成器

> skill id：fullstack-qa-weekly-report
> 当前版本：1.0.0
> 产品状态：可用
> 所属分类：互联网研发 / 研发角色 / 全栈测试 / 周报

## 产品定位

周报围绕风险是否被验证和版本是否可安全发布，不用用例执行数量制造质量感。

## 解决的问题

- 周报只罗列执行多少用例和发现多少 bug
- 高风险未测但整体通过率看起来很高
- 测试环境问题、产品缺陷和发布风险混在一起

## 适用场景

- 个人或团队测试周报
- 版本发布质量周报
- 从测试平台、缺陷和生产验证记录重建周报

## 不适用范围与边界

- 不得用通过率掩盖未覆盖高风险行为
- 不得把环境故障或数据问题自动算作产品缺陷
- 不得把缺陷关闭等同根因消除和回归完成
- 不得编造 release ready、覆盖率、缺陷趋势或生产质量结论

## 输入

- 报告周期、版本与发布范围
- 风险、测试、缺陷、环境和发布证据
- 可选上周周报、质量指标和下周版本计划

## 输出

- 全栈测试周报 Markdown
- 风险覆盖与发布 gate 摘要
- 缺陷、阻塞和下周验证计划

## 工作原理

1. 锁定报告周期、时区、版本/项目范围、发布节点和是否需要对比上周
2. 盘点需求、风险矩阵、测试计划、执行记录、自动化、缺陷、环境、发布和生产证据
3. 按用户旅程和风险领域聚合前端、API、后端、数据、性能、安全与兼容验证
4. 区分已覆盖、部分覆盖、未覆盖、阻塞和不适用，并记录可信 oracle
5. 汇总缺陷时保留严重度、影响、状态、归属版本和复现证据，避免只报数量
6. 评估 release gate、迁移、回归、生产 smoke、监控和回滚验证
7. 质量指标必须保留分母、口径、周期和历史可比性，说明样本不足
8. 把环境、数据、依赖和权限问题与产品缺陷分开
9. 输出当前质量结论、发布建议、剩余风险和需要决策的阻塞项
10. 下周计划以关闭高风险证据缺口和版本 gate 为主，不机械追求更多用例

## 技能包组成

- SKILL.md：AI 周报生成入口。
- references/weekly-quality-evidence.md：状态和指标证据检查。
- assets/qa-weekly-template.md：中文周报模板。
- eval/acceptance.md：真实性和可比性验收。
- README.md：中文产品文档与版本记录。

## 使用方式

1. 明确周期、时区、项目/版本和汇报对象。
2. 提供工单、PR、测试、缺陷、部署或发布证据。
3. 让 AI 按模板生成并使用验收清单检查。

## 验收标准

- 版本、周期、风险范围和 gate 是否明确
- 覆盖结论是否映射到风险和 oracle
- 缺陷数量是否保留严重度、影响和状态口径
- 发布建议是否包含剩余风险、阻塞和回滚/生产验证
