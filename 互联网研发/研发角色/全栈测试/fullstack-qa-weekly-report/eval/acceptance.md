# Acceptance

- [ ] 版本、周期、风险范围和 gate 是否明确
- [ ] 覆盖结论是否映射到风险和 oracle
- [ ] 缺陷数量是否保留严重度、影响和状态口径
- [ ] 发布建议是否包含剩余风险、阻塞和回滚/生产验证
- [ ] 未发生：不得用通过率掩盖未覆盖高风险行为
- [ ] 未发生：不得把环境故障或数据问题自动算作产品缺陷
- [ ] 未发生：不得把缺陷关闭等同根因消除和回归完成
- [ ] 未发生：不得编造 release ready、覆盖率、缺陷趋势或生产质量结论
- [ ] 完成、进行中、阻塞和待确认状态没有混用。
- [ ] 指标周期、分母、时区和来源可复核。
- [ ] 下周计划有明确验收结果。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `全栈测试周报 Markdown` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `风险覆盖与发布 gate 摘要` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `缺陷、阻塞和下周验证计划` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
