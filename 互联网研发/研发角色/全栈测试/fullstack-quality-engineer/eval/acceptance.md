# Acceptance

- [ ] 每个高风险行为是否有测试层和可信 oracle
- [ ] 身份、数据、locale、时区和环境差异是否覆盖
- [ ] 迁移、异步、第三方和失败恢复是否验证
- [ ] 发布 gate、生产 smoke 和回滚触发是否可执行
- [ ] 未发生：不得用端到端用例替代所有低层精确验证
- [ ] 未发生：不得只测接口 200 或页面出现而忽略数据和副作用
- [ ] 未发生：不得在共享环境制造不可清理或敏感测试数据
- [ ] 未发生：不得因测试通过忽略观测、迁移和回滚不可验证
- [ ] 每项完成结论能追溯到跨层证据。
- [ ] 发布、回滚和生产验证已明确。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `全栈风险与覆盖矩阵` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `跨层测试结果和缺陷证据` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `发布 gate 与阻塞项` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `生产验证、回滚和数据修复方案` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
