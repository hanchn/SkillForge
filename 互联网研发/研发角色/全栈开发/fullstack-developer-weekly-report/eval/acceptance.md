# Acceptance

- [ ] 周期、时区和项目范围是否明确
- [ ] 每项完成事项是否有合并/发布/验证证据
- [ ] 同一目标的跨层改动是否合并表达
- [ ] 风险和下周计划是否包含依赖与可验收结果
- [ ] 未发生：不得把 commit、PR 或代码行数直接等同业务产出
- [ ] 未发生：不得把已编码、已提测、已合并和已上线混为完成
- [ ] 未发生：不得编造进度百分比、发布日期、指标改善或责任归属
- [ ] 未发生：对外周报避免泄露凭据、内部敏感架构和不必要技术噪音
- [ ] 完成、进行中、阻塞和待确认状态没有混用。
- [ ] 指标周期、分母、时区和来源可复核。
- [ ] 下周计划有明确验收结果。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `全栈开发周报 Markdown` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `证据与状态对照表` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `风险、依赖和下周验收计划` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
