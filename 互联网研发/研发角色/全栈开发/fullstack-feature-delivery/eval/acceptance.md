# Acceptance

- [ ] 需求、权限、状态与不变量是否跨层一致
- [ ] API 和 schema 是否允许发布期间新旧版本共存
- [ ] 每层是否有匹配风险的测试与观测
- [ ] 部署、迁移、回滚和生产验证是否完整
- [ ] 未发生：不得绕过仓库现有边界重新搭建独立技术栈
- [ ] 未发生：不得伪造 API、数据、权限或产品决策
- [ ] 未发生：不得只验证某一层而宣称端到端完成
- [ ] 未发生：不得在没有迁移、回滚和生产验证时交付高风险变更
- [ ] 每项完成结论能追溯到跨层证据。
- [ ] 发布、回滚和生产验证已明确。


## 交付物专项验收

- [ ] 已询问使用者现有数据、系统、模板、词库、规则、优先级和不可改变项
- [ ] 已标明时效事实的数据截止、采集时间、来源、版本和适用范围
- [ ] `端到端实现` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `跨层变更和依赖说明` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `测试、迁移、发布与回滚证据` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] `生产验证与剩余风险` 已完整交付，并有下游使用者、验收证据和剩余风险
- [ ] 已覆盖正常路径、关键异常、暂停条件、人工升级、补偿或回滚
- [ ] 最终结论明确事实、假设、未知项、owner、审批和下一次复核时间
