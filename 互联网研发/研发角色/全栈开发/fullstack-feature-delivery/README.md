# 全栈开发资深专家

> skill id：fullstack-feature-delivery
> 当前版本：1.0.2
> 产品状态：可用
> 所属分类：互联网研发 / 研发角色 / 全栈开发

## 产品定位

按用户可验证的垂直切片交付，不把前端完成、接口完成或代码合并误认为功能完成。

## 解决的问题

- 前端和后端分别完成但契约、权限和状态不一致
- 数据库变更与应用发布顺序造成停机
- 代码合并后缺少真实用户流程和生产验证

## 适用场景

- 实现跨前端、接口和数据库的新功能
- 修复需要多层协同的业务缺陷
- 把一个需求拆成可部署、可回滚的垂直切片

## 不适用范围与边界

- 不得绕过仓库现有边界重新搭建独立技术栈
- 不得伪造 API、数据、权限或产品决策
- 不得只验证某一层而宣称端到端完成
- 不得在没有迁移、回滚和生产验证时交付高风险变更

## 输入

- 产品需求、仓库和目标环境
- 现有 API、数据、权限与发布约束
- 可选设计、测试账号和验收指标

## 输出

- 端到端实现
- 跨层变更和依赖说明
- 测试、迁移、发布与回滚证据
- 生产验证与剩余风险

## 工作原理

1. 读取仓库规则、产品需求、现有架构、环境、命令、发布方式和相邻功能
2. 把需求转成角色、流程、状态、权限、数据、不变量、验收和非目标
3. 建立从 UI 到 API、领域逻辑、数据库、事件、任务、观测和部署的影响图
4. 先锁定 API 和数据兼容边界，再按可部署切片规划前端、后端和迁移顺序
5. 实现前端正常、加载、空、错误、权限、重试和并发状态
6. 实现服务端验证、授权、不变量、幂等、事务、失败恢复和审计
7. 用 expand-migrate-contract 处理数据库变化，保证新旧版本共存和对账
8. 加入结构化日志、指标、trace、feature flag、告警和运营可见性
9. 运行单元、组件、契约、集成、端到端和迁移验证，避免重复而无目的的覆盖
10. 制定部署、灰度、回滚和生产验证，报告每层改动、证据和剩余风险

## 技能包组成

- SKILL.md：AI 执行入口。
- references/fullstack-delivery-checklist.md：跨层专业检查表。
- assets/vertical-slice-plan.md：端到端追溯模板。
- eval/acceptance.md：验收标准。
- README.md：中文角色产品文档和版本记录。

## 使用方式

1. 提供需求、仓库、环境和发布约束。
2. 让 AI 读取 SKILL.md 与检查表。
3. 使用矩阵保持跨层追溯并在交付前验收。

## 验收标准

- 需求、权限、状态与不变量是否跨层一致
- API 和 schema 是否允许发布期间新旧版本共存
- 每层是否有匹配风险的测试与观测
- 部署、迁移、回滚和生产验证是否完整
