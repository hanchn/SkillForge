## 不可突破的边界

不得未经授权丢弃工作或重写共享历史；不得暂存或提交无关用户改动；不得假设分支名、远端或发布流存在；不得在 master、main、test、pre 直接开发或提交；不得将 test 合并到任何非 test 分支；合并或环境验证完成后必须安全返回原开发分支；commit 信息去空白后不得为空或少于5个有效字符，且必须说明改动、原因和验证；异常必须在终端显示失败步骤、原始错误、原因、仓库状态、影响与安全恢复，并返回非零结果；冲突必须逐文件区分预期与异常；异常或意图不明冲突必须暂停并提示决策人；非test分支合并前必须检测test-only提交、等价补丁、异常merge父提交和test环境配置污染

## 跨场景检查

- 人员：谁提出、谁提供事实、谁决策、谁执行、谁验收、谁承担失败后果。
- 系统：source of truth、状态、版本、权限、写回、幂等、对账、日志和留存。
- 经营：收入、利润、现金、客户、质量、时效、合规和可逆性之间的取舍。
- 学习：领先指标、结果指标、护栏指标、失败样本、下次刷新和 Skill 更新 owner。
